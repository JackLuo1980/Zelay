# 端口复用功能指南

## 📖 概述

端口复用功能可以解决 **NAT 端口数量不足** 的问题，特别适用于 IX 等端口资源有限的场景。

通过端口复用，多个不同的转发目标可以共享同一个监听端口，大大减少所需的端口数量。

## 📝 配置步骤

### 第一步：配置 IX 服务器（端口复用服务端）

> 以 IX 地址 `138.252.163.123` 为例

1. 在管理面板创建 Agent 并在 IX 服务器上部署
2. 创建转发规则，**勾选「启用端口复用」**

![IX 服务端配置](https://raw.githubusercontent.com/enp6/Zelay/main/m1.jpg)

**配置说明：**
| 字段 | 说明 |
|------|------|
| 监听地址 | IX 服务器监听的地址和端口（如 `::` 监听所有接口） |
| 启用端口复用 | ✅ 必须勾选 |
| 远程地址（可选） | 非协议流量的默认转发目标，可留空 |

---

### 第二步：配置前置服务器（端口复用客户端）

1. 在管理面板创建 Agent 并在前置服务器上部署
2. 创建转发规则，**勾选「通过端口复用连接」**
3. 填写 IX 服务器地址和最终目标地址

![前置服务器配置](https://raw.githubusercontent.com/enp6/Zelay/main/m2.jpg)

**配置说明：**
| 字段 | 说明 |
|------|------|
| 监听地址 | 前置服务器监听的地址和端口 |
| 通过端口复用连接 | ✅ 必须勾选 |
| 远程地址 | IX 服务器的地址和端口（如 `138.252.163.123:10000`） |
| 最终目标地址 | 实际要访问的目标服务地址（如 `192.168.1.100:8080`） |

---

## 🎯 使用场景示例

### 场景：一个 IX 端口转发多个游戏服务器

```
前置 A (监听 :25565) ──┐
                      │     IX 服务器
前置 B (监听 :25566) ──┼──── (:10000) ────┬── MC服务器1 (192.168.1.10:25565)
                      │                  ├── MC服务器2 (192.168.1.11:25565)
前置 C (监听 :25567) ──┘                  └── MC服务器3 (192.168.1.12:25565)
```

**优势：** IX 只需要开放 **1 个端口**，即可转发到 **多个不同目标**

## **TCP/UDP 支持**：端口复用同时支持 TCP 和 UDP 协议
